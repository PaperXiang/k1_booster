# Lissajous 找球扫描调整报告

## 修改目标

解决丢球后找球扫描视角偏低、身体转身不持续的问题。让丢球搜索时头部 Lissajous 扫描和身体转身同时进行，并在短暂疑似看到球时不立刻完全停转。

## 改动文件

- `src/brain/src/brain_tree.cpp`

## 关键逻辑变化

- `CamFindBall` 中 Lissajous 参数调整：
  - `bodyVtheta`: `1.2` -> `1.0`
  - `headPeriod`: `2.4` -> `2.8`
  - `pitchCenter`: `0.30` -> `0.22`
  - `pitchAmp`: `0.34` -> `0.22`
- 扫描中心抬高，减少一直低头看近处的问题。
- 降低 pitch 振幅，避免扫描周期中过度向下。
- 丢球搜索阶段仍持续发送身体转身命令。
- 刚检测到球但尚未达到稳定确认次数时，不再马上完全停转，而是以 `bodyVtheta * 0.5` 慢速保持转身，减少视觉抖动导致“停住不找”的情况。
- 只有球检测达到稳定确认条件后，才停止身体转动。

## 内容审查结论

- 找球核心仍然是 `CamFindBall` 的 `lissajous_v2`。
- 本次没有改行为树结构，只改找球节点内部扫描参数和检测到球时的停转策略。
- XML 解析检查发现当前 `game.xml` 存在既有未转义字符问题，位置约在第 32 行，和本次 `CamFindBall` 改动无关。
